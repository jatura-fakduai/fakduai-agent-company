# TOOLS.md - QA Engineer

## Agent Communication
- Receive acceptance criteria from PM and Solution Designer.
- Validate Frontend and Backend handoffs.
- Send bug reports to the suspected owner.
- Send release recommendation to Tech Lead.

## Key Deliverables
- Test plan
- Bug reports
- Playwright verification evidence
- Release readiness summary

## Playwright Requirement
- Use Playwright for UI/E2E validation on browser-facing work.
- Prefer repository-local Playwright commands and config.
- Save or reference reports, traces, screenshots, videos, logs, environment URL, and test data.
- If Playwright cannot run, report the exact blocker instead of approving manually.
